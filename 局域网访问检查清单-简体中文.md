# 局域网访问检查清单（简体中文）

## 一、先确认你本机能打开

先在运行项目的这台电脑打开：

- `http://127.0.0.1:3000`

如果你自己电脑都打不开，先不要测局域网。

---

## 二、启动方式必须正确

请在项目目录启动：

```bat
cd /d C:\Users\AEboli\Documents\Codex
npm run dev -- --hostname 0.0.0.0
```

或者直接双击：

- `C:\Users\AEboli\Documents\Codex\启动开发版.bat`
- `C:\Users\AEboli\Documents\Codex\启动正式版.bat`

### 关键点

必须带上：

```text
--hostname 0.0.0.0
```

否则其他局域网设备通常访问不到。

---

## 三、查看你电脑的局域网 IP

你可以双击：

- `C:\Users\AEboli\Documents\Codex\查看局域网地址.bat`

或者手动运行：

```bat
ipconfig
```

找到 IPv4 地址，比如：

```text
192.168.1.23
```

那么其他设备访问：

```text
http://192.168.1.23:3000
```

---

## 四、如果局域网其他设备访问失败

按顺序检查：

### 1）是否在同一个局域网

例如：
- 都连接同一个 Wi-Fi
- 或都在同一路由器下

### 2）Windows 防火墙是否拦截 Node.js

如果第一次启动时弹出了防火墙提示：

- 请选择“允许访问”
- 最好勾选专用网络

如果没弹窗，可以手动去 Windows 防火墙里放行 `node.exe`

### 3）是否真的使用了局域网地址访问

错误示例：

```text
http://127.0.0.1:3000
```

这个地址只能你自己电脑访问。

正确是：

```text
http://192.168.x.x:3000
```

### 4）端口是否被占用或被拦截

如果 3000 被占用，可以换成 3001：

```bat
npm run dev -- --hostname 0.0.0.0 --port 3001
```

然后其他设备访问：

```text
http://你的局域网IP:3001
```

### 5）路由器是否禁止设备互访

有些路由器会开启：
- AP 隔离
- 客户端隔离
- 访客网络隔离

如果开了，设备之间不能互相访问。

---

## 五、如果页面能打开，但生成功能失败

这通常不是局域网问题，而是 API 设置问题。

请检查：

- 设置页是否填了 Gemini API Key
- 默认文本模型是否为 `gemini-2.5-flash`
- 默认图像模型是否为 `gemini-2.5-flash-image`
- 一次任务生成数量是否过大

第一次建议最小测试：

- 1 张图
- 1 种图片类型
- 1 个比例
- 1 个分辨率
- 数量 = 1
