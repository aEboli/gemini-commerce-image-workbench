# 全局协作规则

## 1. 阶段性进展同步

- 每完成一个小阶段任务后，必须更新 `doc/进展记录.md`。
- 每次更新至少包含：阶段目标、已完成事项、涉及文件、待办事项或风险。
- 如果某项任务需要单独跟踪，可以在 `doc/` 目录下新增专题进展文档，并在 `doc/进展记录.md` 中补充索引。

## 2. 会话上下文提醒

- 当判断当前会话上下文接近限制、继续展开可能影响连续性时，必须主动提醒用户。
- 提醒时必须同步输出一份可续接总结，便于在新会话中继续工作。
- 续接总结至少包含：当前目标、已完成内容、未完成事项、建议下一步、关键文件或命令。

## 3. 使用说明与 PRD 交付

- 当本次对话产物需要交付给其他人使用或维护时，在 `Readme/` 目录下补充对应使用说明。
- 使用说明文件名需要明确标注对象，推荐格式为 `使用说明-<对象>.md`。
- 同步在 `Readme/` 目录下补充对应 PRD，推荐格式为 `PRD-<对象>.md`。
- 使用说明和 PRD 需要在文档开头写明其对应的模块、功能或项目对象。
- 新增或更新使用说明、PRD 后，需要在 `doc/进展记录.md` 中记录新增文件名及其用途。

## 4. 目录约定

- `doc/`：持续更新的进展文档与阶段总结。
- `Readme/`：面向协作者的使用说明、PRD 与交付说明。
